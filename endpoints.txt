# SCMS Web API Endpoints
# Base URL: http://localhost:5140

## Auth
- `POST   /api/Auth/register`
- `POST   /api/Auth/login`
- `POST   /api/Auth/refresh`

## Dashboards
- `GET    /api/Dashboards/dashboard`               # Owner-only admin dashboard
- `GET    /api/Dashboards/patient-dashboard`        # Patient dashboard (auth required)

## Appointments
- `POST   /api/Appointments`                        # Book appointment
- `GET    /api/Appointments`                        # List (with filters: startDate, endDate, status, patientId, pagination)
- `PATCH  /api/Appointments/{id}/status`            # Owner: update status
- `POST   /api/Appointments/{id}/reschedule`        # Owner: reschedule
- `GET    /api/Appointments/{id}/queue-status`      # Get patient queue status
- `POST   /api/Appointments/call-next`              # Owner: call next in queue

## Patients
- `POST   /api/Patients`                            # Add patient profile
- `GET    /api/Patients`                            # List profiles (pagination)
- `GET    /api/Patients/patients/{id}`              # Get patient by ID
- `GET    /api/Patients/{id}/history`               # Patient visit history
- `GET    /api/Patients/{id}/summary`               # Medical summary (JSON)
- `GET    /api/Patients/{id}/summary/html`          # Medical summary (HTML)
- `GET    /api/Patients/{id}/summary/pdf`           # Medical summary (PDF download)

## Prescriptions
- `POST   /api/Prescriptions`                      # Owner: create prescription
- `GET    /api/Prescriptions`                       # List (with patientId filter, pagination)
- `GET    /api/Prescriptions/prescriptions/{id}`    # Get prescription details
- `POST   /api/Prescriptions/templates`             # Owner: save template
- `GET    /api/Prescriptions/templates`             # Owner: list templates (diseaseId filter, pagination)
- `DELETE /api/Prescriptions/templates/{id}`        # Owner: delete template
- `GET    /api/Prescriptions/{id}/pdf`              # Download prescription PDF

## Medicines
- `GET    /api/Medicines`                           # List medicines
- `POST   /api/Medicines`                           # Owner: create medicine
- `PUT    /api/Medicines/{id}`                      # Owner: update medicine
- `DELETE /api/Medicines/{id}`                      # Owner: delete medicine
- `GET    /api/Medicines/categories`                # List medicine categories
- `POST   /api/Medicines/quarantine-expired`        # Owner: quarantine expired batches
- `GET    /api/Medicines/alerts`                    # Owner: inventory alerts (low stock, expiry)
- `GET    /api/Medicines/batches`                   # Owner: list batches (with filters)
- `GET    /api/Medicines/batches/{id}`              # Owner: get batch details
- `POST   /api/Medicines/batches`                   # Owner: add new batch
- `PUT    /api/Medicines/batches/{id}`              # Owner: update batch
- `DELETE /api/Medicines/batches/{id}`              # Owner: delete batch

## Diseases
- `GET    /api/Diseases`                            # List diseases (with query, pagination)
- `POST   /api/Diseases`                            # Create disease
- `PUT    /api/Diseases`                            # Update disease
- `DELETE /api/Diseases/{id}`                       # Delete disease

## Payments
- `GET    /api/Payments`                            # List payments
- `POST   /api/Payments/gateway-callback`           # Payment gateway callback
- `POST   /api/Payments/manual-proof`               # Upload manual payment proof
- `POST   /api/Payments/{id}/approve`               # Owner: approve payment
- `GET    /api/Payments/{id}/invoice/pdf`            # Download invoice PDF

## FollowUps
- `GET    /api/FollowUps`                           # List follow-ups (pagination)
- `POST   /api/FollowUps`                           # Owner: create follow-up
- `POST   /api/FollowUps/{id}/complete`             # Owner: mark complete

## Notifications
- `GET    /api/Notifications`                       # List notifications (with includeAll, pagination)
- `POST   /api/Notifications`                       # Owner: create notification
- `POST   /api/Notifications/{id}/read`             # Mark notification as read

## Reports (Owner-only)
- `GET    /api/Reports/appointments`                # Appointment report (JSON) — ?reportType=daily&date=
- `GET    /api/Reports/appointments/pdf`            # Appointment report (PDF download)
- `GET    /api/Reports/revenue`                     # Revenue report (JSON) — ?reportType=daily&date=
- `GET    /api/Reports/revenue/pdf`                 # Revenue report (PDF download)
- `GET    /api/Reports/patients`                    # Patient list report (JSON)
- `GET    /api/Reports/patients/pdf`                # Patient list report (PDF download)
- `GET    /api/Reports/medicine-stock`              # Medicine stock report (JSON)
- `GET    /api/Reports/medicine-stock/pdf`          # Medicine stock report (PDF download)
- `GET    /api/Reports/follow-ups`                  # Follow-up report (JSON) — ?reportType=all&date=&status=
- `GET    /api/Reports/follow-ups/pdf`              # Follow-up report (PDF download)
- `GET    /api/Reports/business-summary`            # Monthly business summary (JSON) — ?date=
- `GET    /api/Reports/business-summary/pdf`        # Monthly business summary (PDF download)

## MCP (Model Context Protocol — Owner-only)
- `GET    /api/mcp/tools`                           # List available MCP tools
- `POST   /api/mcp/tools/call`                      # Call an MCP tool
- `POST   /api/mcp/chat`                            # AI chat interaction

## SignalR Hubs (Real-time WebSocket)
- `/hubs/queue`                                     # QueueHub — appointment queue updates
- `/hubs/notifications`                             # NotificationsHub — live notification delivery
# Auth: JWT via `access_token` query parameter for hub connections
